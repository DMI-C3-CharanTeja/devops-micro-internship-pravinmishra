### Answers

#### Question 1 — What is an AWS account, and why do you need it at this stage?

An AWS account is your personal or organization's account that provides access to Amazon Web Services (AWS), allowing you to create, manage, and use cloud resources such as virtual servers (EC2), storage (S3), databases, networking, and many other services.

At this stage, you need an AWS account because it allows you to:

Launch and manage an EC2 instance (virtual Linux server) for hands-on practice.
Deploy applications, such as a React app with Nginx.
Practice Linux administration, Bash scripting, and DevOps tasks in a real cloud environment.
Learn cloud infrastructure in a safe, isolated environment without affecting production systems.
Gain practical experience with AWS services that are widely used in the industry.

In short, an AWS account provides the cloud environment needed to complete the assignments and gain real-world DevOps experience.

---

#### Question 2 — What is AWS Free Tier, and how long does it last?



AWS Free Tier is a program that lets new AWS customers use selected AWS services at no cost, within specified monthly usage limits. It is designed to help you learn and experiment with AWS.

For most commonly used services, such as Amazon EC2, the 12-month Free Tier begins on the date you create your AWS account. During this period, you can use eligible resources up to the monthly free limits (for example, eligible EC2 instance hours and storage). If you exceed those limits or use non-free services, you will be charged according to AWS pricing.

In summary:

What is it? A set of free usage allowances for eligible AWS services.
How long does it last? Most introductory Free Tier benefits last 12 months from the date you create your AWS account, while some AWS services also offer "Always Free" or short-term free trials with different terms.

#### Question 3 — Name three AWS Free Tier services and their free usage limits.

The uploaded assignment does not list AWS Free Tier services or their usage limits. It only states that you need a running Ubuntu EC2 instance as a prerequisite.

Using general AWS knowledge, here are three common AWS Free Tier services and their limits (for new AWS customers):

Amazon EC2
750 hours per month of eligible EC2 instance usage (within Free Tier limits) for the first 12 months.
Amazon S3
5 GB of Standard Storage.
Includes limited requests and 15 GB of data transfer out per month (Free Tier limits).
Amazon RDS
750 hours per month of a Single-AZ eligible database instance.
Includes 20 GB of database storage and 20 GB of backup storage for the first 12 months.
